Corporate Information Security Policy (CISP)
Section: Technical Access Controls
Last Modified: May 2026

All employees, contractors, and third-party vendors must authenticate through the corporate Identity Provider (IdP) using Multi-Factor Authentication (MFA) before accessing internal environments. 

Production networks are strictly isolated from development and staging environments. Access to production AWS resources is managed through the AWS Security Token Service (STS), utilizing short-lived IAM assume-role credentials that expire after a maximum of one hour. 

Data at rest within all S3 buckets and EBS volumes must be encrypted using customer-managed keys via AWS KMS. Data in transit must use TLS 1.3 encryption protocols. 

To maintain network boundaries, Security Groups are configured with a default-deny posture. Only explicit ports required for application execution (such as Port 443 for HTTPS traffic) are open. ICMP traffic is entirely disabled across public-facing subnets to avoid network mapping attempts by malicious actors. Regular automated vulnerability scans are handled by AWS Trusted Advisor.
